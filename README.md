# vcno-rbno

Code for the paper
[Variationally correct operator learning: Reduced basis neural operator with a posteriori error estimation](https://arxiv.org/abs/2512.21319). 

This work develops a variationally correct operator learning framework by constructing first-order system least-squares (FOSLS) objectives whose values are provably equivalent to the solution error in PDE-compliant norms. We demonstrate this framework on stationary diffusion and linear elasticity, incorporating mixed Dirichlet-Neumann boundary conditions via variational lifts to preserve norm equivalence without inconsistent penalties. To ensure the function space conformity required by the FOSLS loss, we propose a Reduced Basis Neural Operator (RBNO). The RBNO predicts coefficients for a pre-computed, conforming reduced basis, thereby ensuring variational stability by design while enabling efficient training.
 
## Repository layout

- `configs/`: YAML configuration files for meshes, function spaces, and reduced-basis settings.
- `data_generation/`: PDE solvers, probability-measure generation, reduced-basis construction, reduced loss weights, and reference minimizers.
- `scripts/`: shell entry points for directory creation, data download, and data generation pipelines.
- `results/`: generated data (not committed to Git).
- `models/`: neural network model definitions used by notebooks.
- `analysis/`: analysis notebooks and saved figures.
- `train/`: training and evaluation notebooks.
- `utils/`: shared IO, plotting, MPI, and helper utilities.

## Environment setup

Create the environment from pinned lock files:

```bash
cd /path/to/vcno-rbno
conda create -n fenicsx-test --file environment/fenicsx-linux-64-explicit.txt -y
conda activate fenicsx-test
conda install -n fenicsx-test pip -y
python -m pip install -r environment/fenicsx-pip-requirements.txt
```

Set OpenMP threads to 1 to avoid CPU oversubscription in MPI workflows:

```bash
echo 'export OMP_NUM_THREADS=1' >> ~/.bashrc
source ~/.bashrc
```

Packages and their versions used in the paper runs:
- Python `3.13`
- DOLFINx `0.9.0`
- scifem `0.7.0`
- PyTorch `2.6.0+cu124`


## Problem setups and data generation

We consider the following problem setups:
- `poisson_setup1` (Heat Conduction)
- `poisson_setup2` (Darcy Flow)
- `elasticity` (Linear Elasticity)


Data generation: create directories for saving results

```bash
cd /path/to/vcno-rbno
bash scripts/create_results_directories.sh
```

Download the parameter data for Poisson setup 2 and elasticity (generated using hIPPYlib 3.1.0) from Hugging Face into `results/` (parameter data for Poisson setup 1 is generated by the `poisson_permeability_setup1.py`):

```bash
bash scripts/download_results.sh https://huggingface.co/datasets/yqiu327/vcno-rbno-results/resolve/main
```


Generate remaining derived data (PDE solutions, POD basis, reduced weight matrices, etc):

```bash
bash scripts/poisson_least_squares_setup1_data_generation.sh
bash scripts/poisson_least_squares_setup2_data_generation.sh
bash scripts/elasticity_least_squares_data_generation.sh
```

The data-generation scripts launch 50 MPI ranks by default (`mpirun -n 50`). Reduce or increase `-n` to match your available CPU cores.

## Analysis

Analysis notebooks are grouped by problem:
- `analysis/poisson_setup1/`
- `analysis/poisson_setup2/`
- `analysis/elasticity/`

These notebooks cover visualization, low-rank approximation, trailing eigenvalues, reduced-basis loss, reconstruction error, and wall-clock comparisons.

## Model training and inference

Training notebooks are grouped by problem:
- `train/poisson_setup1/`
- `train/poisson_setup2/`
- `train/elasticity/`

Models covered:
- Reduced Basis Neural Operator (RBNO)
- Fourier Neural Operator (FNO)
- Principal Component Analysis Network (PCA-Net)

Notebook naming patterns:
- RBNO, physics (FOSLS) loss only: `*_rbno_physics_loss.ipynb`
- RBNO, POD data loss only: `*_rbno_pod_data_loss.ipynb`
- RBNO, physics + POD data losses: `*_rbno_physics_loss_pod_data_loss.ipynb`
- FNO: `*_fno.ipynb`
- PCA-Net (basis computed from grid-point evaluations): `*_pcanet.ipynb`
- PCA-Net (basis computed from degrees of freedom): `*_pcanet_new_basis.ipynb`
